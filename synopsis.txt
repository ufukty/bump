NAME:
    bump - version safely.

SYNOPSIS:
    bump <command> [--args]

COMMANDS:
    major [--force]
      Issues a semantic version tag by incrementing the major label of the current version by one (e.g., v1.2.3 to v2.0.0). Initializes at v1.0.0. Landing on v1.0.0 requires the --force flag and otherwise fails in order to prevent accidentally committing to backwards-compatibility guarantees. Further, consider postponing v1 and search '0ver' on web.

    minor
      Issues a semantic version tag by incrementing the minor label of the current version by one (e.g., v1.2.3 to v1.3.0). Initializes at v0.1.0.

    patch
      Issues a semantic version tag by incrementing the patch label of the current version by one (e.g., v1.2.3 to v1.2.4). Initializes at v0.0.1.

    alpha
    alpha ( major [--force] | minor | patch )
    alpha finalize [--force]
      Initiates, iterates, and finalizes an alpha track. An alpha track is a successive series of pre-release version tags that prefixed with their target version. Initiating one requires a label for determining the target version. E.g., 'alpha minor' on v1.2.3 creates v1.3.0-alpha.1 and 'alpha major' creates v2.0.0-alpha.1. Both targeting and landing on v1.0.0 requires --force for the same reason as the major command.
